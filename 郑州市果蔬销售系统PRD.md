## 一、文档概述

### 1.1 文档目的

本文档旨在详细描述郑州市果蔬在线销售系统的产品需求，为开发团队提供清晰的功能规格说明和技术实现指导。文档涵盖系统背景、用户角色、功能需求、非功能需求、技术架构、数据模型及开发计划等内容，确保开发过程中各方对产品需求有统一的理解和认识。

### 1.2 产品范围

本系统是一个面向郑州市本地果蔬销售的综合性电商平台，集成了在线交易、信息展示、社区互动、全流程溯源及AI智能问答等核心功能。系统服务于四类用户角色：系统管理员、商家、注册用户和游客，通过前后端分离的技术架构，为用户提供便捷、透明、可信赖的果蔬购买体验。

### 1.3 术语定义

| 术语 | 定义说明 |
|------|---------|
| 果蔬圈 | 系统内的社区互动模块，商家可发布图文动态展示种植过程与农场环境 |
| 全流程溯源 | 从产地、种植、采收、检测到物流的完整信息追踪体系 |
| AI果蔬专家 | 基于人工智能技术的智能问答模块，提供果蔬知识科普与咨询服务 |
| B/S架构 | 浏览器/服务器架构，用户通过浏览器访问系统服务 |
| RESTful API | 一种符合REST风格的Web API设计规范，用于前后端数据交互 |

## 二、产品背景与目标

### 2.1 背景分析

随着郑州市居民生活水平的显著提高与健康消费理念的日益普及，市民对新鲜、安全的优质果蔬需求持续增长，并对产品的可追溯性提出了更高要求。传统的果蔬销售模式，如农贸市场与实体超市，在信息透明度、购买便捷度、供应链追溯等方面存在局限，难以完全满足现代消费者的期望。

与此同时，数字经济的蓬勃发展为传统零售业的转型升级提供了强大动力。在此背景下，开发一个集线上交易、信息展示、社区互动与全流程溯源于一体的郑州市果蔬在线销售系统显得尤为迫切。该系统旨在通过信息化手段整合本地优质果蔬供应链资源，为消费者提供一个便捷、透明、可信赖的购买平台，同时助力本地果蔬商户拓展销售渠道，提升品牌影响力，推动郑州市果蔬产业的数字化发展。

### 2.2 产品目标

#### 2.2.1 核心目标

1. **构建便捷交易平台**：实现果蔬商品的在线展示、智能搜索、分类筛选、购物车管理、在线下单与支付等功能，简化购买流程，提升交易效率。

2. **增强消费信任**：通过"果蔬圈"功能，让商户分享种植过程与环境，增强消费者信任；通过全流程溯源系统，使消费者能够便捷查询果蔬的产地、种植、采收、检测及物流信息。

3. **提供智能服务**：引入AI果蔬专家智能问答模块，为消费者提供及时的果蔬知识科普、存储建议、食用方法等咨询服务，提升用户体验。

4. **降低运营成本**：为果蔬供应商提供便捷的商品信息管理、库存更新、订单处理、销售数据分析等功能，降低其运营管理成本。

#### 2.2.2 预期效益

- **消费者层面**：信息透明化增强消费者信心与满意度，智能服务丰富购物体验
- **商家层面**：拓展销售市场，降低渠道成本，提升品牌形象，数据分析辅助经营决策
- **社会层面**：推动郑州市果蔬销售模式的数字化转型，促进本地农产品电商发展，构建高效、透明、安全的农产品流通体系

## 三、用户角色分析

### 3.1 角色概览

系统共设计四类用户角色，各角色的定位与权限如下表所示：

| 角色类型 | 角色描述 | 主要权限范围 |
|---------|---------|-------------|
| 系统管理员 | 平台运营管理人员 | 全局管理、审核监管、数据维护 |
| 商家 | 入驻平台的果蔬供应商 | 店铺运营、商品管理、订单处理 |
| 用户 | 注册的普通消费者 | 浏览购物、互动交流、订单管理 |
| 游客 | 未注册的访问者 | 有限浏览、引导注册 |

### 3.2 角色详细说明

#### 3.2.1 系统管理员

**角色定位**：系统管理员是平台的最高权限持有者，负责整体运营管理、内容监管和系统维护工作。管理员需要具备较强的业务理解能力和数据分析能力，能够及时发现并处理平台运营中的各类问题。

**核心职责**：
- 用户与权限管理：管理商家和用户账户，分配角色权限
- 入驻审核：审核商家提交的入驻申请及资质文件
- 内容监管：监控平台商品信息，处理违规内容
- 溯源管理：维护溯源信息模板，确保数据完整性
- 订单协调：处理异常订单，协调售后与投诉
- 信息发布：发布平台公告、行业资讯

#### 3.2.2 商家

**角色定位**：商家是平台的商品供应方，包括果蔬种植户、农产品合作社、果蔬批发商等。商家需要完成入驻认证后方可开展销售业务，通过平台展示商品、管理订单、与消费者互动。

**核心职责**：
- 店铺运营：申请入驻、维护店铺基本信息
- 商品管理：发布、编辑、上下架果蔬商品
- 社区运营：发布"果蔬圈"动态，与消费者互动
- 溯源维护：录入商品溯源信息
- 订单履约：处理订单、发货、更新物流
- 数据分析：查看销售数据、经营分析

#### 3.2.3 用户

**角色定位**：用户是平台的核心消费群体，包括郑州市及周边地区的普通消费者。用户通过注册成为平台会员，享受完整的购物、互动和售后服务。

**核心职责**：
- 账户管理：注册登录、维护个人信息
- 商品浏览：搜索、筛选、查看商品详情
- 购物下单：购物车管理、下单支付
- 溯源查询：查看商品溯源信息
- 社区互动：浏览"果蔬圈"、点赞评论
- 智能咨询：使用AI果蔬专家问答
- 订单管理：查看订单、确认收货、申请售后

#### 3.2.4 游客

**角色定位**：游客是未注册系统的访问者，权限较为有限。系统设计引导机制，鼓励游客注册成为正式用户。

**核心权限**：
- 浏览公开信息：首页、部分商品信息、平台公告
- 查看注册流程：被引导注册成为正式用户

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|
| A-01-01 | 用户管理 | 管理商家和用户账户信息，包括查看、编辑、禁用、删除等操作 | P0 |
| A-01-02 | 角色权限管理 | 定义系统角色，分配功能权限，支持细粒度权限控制 | P0 |
| A-01-03 | 系统参数设置 | 配置系统运行参数，如平台名称、联系方式、备案信息等 | P1 |

**详细功能说明**：

**用户管理功能**：
- 支持按用户名、手机号、角色等条件筛选查询用户
- 支持查看用户详细信息，包括注册时间、登录记录、交易记录等

- 支持批量导出用户数据

**角色权限管理功能**：
- 预设系统管理员、商家、用户三类基础角色
- 支持自定义角色创建和权限分配
- 权限控制粒度达到菜单级和按钮级
- 支持角色复制和权限继承

**入驻申请列表**：

**资质审核功能**：
- 在线查看商家上传的资质图片

| 功能编号 | 功能名称 | 功能描述 | 优先级 |

| A-03-02 | 违规处理 | 对违规商品进行下架处理，记录处理原因 | P0 |

**商品列表管理**：

**违规处理功能**：
- 支持单个商品下架操作

**溯源模板管理**：

| A-05-02 | 投诉处理 | 处理用户投诉，协调商家与用户纠纷 | P0 |

### 4.3 商家功能模块

#### 4.3.1 店铺管理

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|
| B-01-01 | 入驻申请 | 提交入驻申请，填写店铺信息，上传资质文件 | P0 |
| B-01-02 | 店铺信息维护 | 编辑店铺名称、简介、Logo、联系方式等 | P0 |
| B-01-03 | 店铺装修 | 自定义店铺首页布局和风格（可选） | P2 |

**详细功能说明**：

**入驻申请流程**：
1. 填写店铺基本信息：店铺名称、经营类型、联系人、联系电话、经营地址
2. 上传资质文件：营业执照、食品经营许可证、法人身份证
3. 阅读并同意平台入驻协议
4. 提交申请，等待审核

**店铺信息维护**：
- 店铺名称：限制20字以内
- 店铺简介：限制500字以内
- 店铺Logo：支持jpg、png格式，建议尺寸200x200
- 联系电话：用于订单通知和客服联系

#### 4.3.2 商品管理

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|---------|---------|---------|--------|

| B-02-02 | 商品编辑 | 编辑已发布商品的信息 | P0 |
| B-02-03 | 商品上下架 | 上架或下架商品 | P0 |
| B-02-04 | 商品分类管理 | 管理店铺商品分类（在系统分类范围内） | P1 |
| B-02-05 | 库存管理 | 管理商品库存，支持库存预警 | P0 |

**详细功能说明**：

**商品发布功能**：
- 商品基本信息：名称、分类、品牌、产地
- 商品规格：重量单位、规格选项
- 商品价格：售价、原价（划线价）
- 商品库存：库存数量、库存预警值
- 商品图片：主图1张，详情图最多9张
- 商品详情：富文本编辑器，支持图文混排
- 溯源信息关联：关联已录入的溯源信息

**商品分类**：
- 系统预设一级分类：蔬菜、水果、菌菇、其他
- 二级分类示例：
  - 蔬菜：叶菜类、根茎类、瓜果类、豆类
  - 水果：浆果类、柑橘类、核果类、瓜果类

|---------|---------|---------|--------|

| B-03-02 | 动态管理 | 编辑、删除已发布的动态 | P0 |
| B-03-03 | 互动管理 | 回复用户评论，管理评论区 | P1 |

**详细功能说明**：

**动态发布功能**：
- 动态标题：限制30字以内
- 动态内容：限制1000字以内
- 图片上传：最多9张图片
- 关联商品：可关联店铺商品，方便用户购买
- 发布范围：公开、仅粉丝可见

**动态内容建议**：
- 种植过程记录：播种、施肥、浇水、生长周期

## 八、接口设计规范

### 8.1 接口规范

#### 8.1.1 基础URL

```
开发环境：http://localhost:8080/api
测试环境：https://test.example.com/api
生产环境：https://api.example.com/api
```

#### 8.1.2 请求规范

- 使用RESTful风格设计接口
- 请求方法：GET（查询）、POST（新增）、PUT（修改）、DELETE（删除）
- 请求头需携带Token：`Authorization: Bearer {token}`
- 请求体格式：JSON

#### 8.1.3 响应规范

**统一响应格式**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1709030400000
}
```

**状态码定义**：

| 状态码 | 说明 |
|-------|------|
| 200 | 操作成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或Token失效 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 0. 中文乱码专项复核（Windows 部署风险）

### 0.1 已完成修复
1. 后端 HTTP 编码强制 UTF-8：`server.tomcat.uri-encoding` + `server.servlet.encoding.*`。
2. JDBC 编码增强：`characterEncoding=UTF-8` + `connectionCollation=utf8mb4_unicode_ci`。
3. 连接初始化：`spring.datasource.hikari.connection-init-sql=SET NAMES utf8mb4 COLLATE utf8mb4_unicode_ci`。
4. Maven 编译/资源编码统一 UTF-8（`pom.xml` 已增加 encoding 配置）。
5. 新增 Windows 防乱码脚本与 SQL：
   - `scripts/windows/import_utf8.cmd`
   - `sql/windows/check_charset.sql`
   - `sql/windows/fix_charset.sql`
   - `docs/windows-utf8-deploy-guide.md`

### 0.2 复核结果
1. `mvn -q -DskipTests clean compile` 通过。
2. `npm run build` 通过。
3. `check_charset.sql` 结果显示：`SCHEMA_CHARSET = utf8mb4 / utf8mb4_unicode_ci`，文本列字符集均为 `utf8mb4`。
4. 接口抽查（`/api/category/list`、`/api/product/list`）返回中文正常，无 `???`。
5. 页面截图中的中文（商品名、分类、产地、菜单）显示正常。

结论：当前复核环境中，中文乱码问题已修复并可复现验证；若另一台 Windows 已存在历史 `???` 数据，仍需按文档重新以 `utf8mb4` 源数据导入。
